# 开发工作流（SDD+TDD 十步流程）

> 本文件描述 orch 插件的完整 10 阶段开发流程。

## 流程概览

```
规范创建 → 测试设计 + 代码设计 → 接口契约 → 任务拆解 → 代码执行 → 测试验证 → 规范归档 → 知识沉淀
```

## 各阶段说明

1. **spec-creation（规范创建）** — 分析需求，产出 BDD 场景、数据模型、业务规则
2. **test-design（测试设计）** — 从规范场景生成测试规格、夹具和模板（与第 3 步并行）
3. **code-design（代码设计）** — 架构设计、数据库模式、接口定义（与第 2 步并行）
4. **api-contract（接口契约）** — 正式化 API 契约并审查一致性（仅全栈模式）
5. **code-task（任务拆解）** — 将设计方案分解为具体的可执行任务
6. **code-execute（代码执行）** — 在独立 git-worktree 中通过子代理逐个实现任务
7. **code-test（测试验证）** — 集成测试、端到端测试、性能验证
8. **spec-archive（规范归档）** — 将验证后的规范合并到主规范仓库
9. **evaluate（效果评估）** — 测试覆盖率、Token 用量、HARD-GATE 质量卡点检查
10. **knowledge-continuum（知识沉淀）** — 提取决策和模式到知识库，实现复利

## HARD-GATE 质量卡点

在第 7→8 步和第 8→9 步之间设有 **HARD-GATE** 关卡：
- 评估覆盖率、测试通过率、规范合规性
- 任一指标未达标则阻塞流水线
- 触发自动补偿流程或通知人工介入
