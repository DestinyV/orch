# E2E测试提示词

指导实现子代理进行端到端(E2E)测试设计。优先使用 `../templates/` 中的 Playwright 配置、工具函数和测试模板。

## E2E测试原则

- **用户视角**：模拟真实用户操作（点击、输入、导航）
- **关键流程**：5-10个核心场景，非全部
- **Happy Path + 失败场景**：正常流程 + 错误处理
- **独立可靠**：不依赖执行顺序 | 显式等待 | 无 flaky

## 测试设计

**场景选择**：核心业务流程（登录→下单→支付）| 关键交互（表单提交、搜索筛选）| 错误恢复（网络失败、超时）

**结构**：`describe('功能') → beforeEach(导航) → test('场景') → expect(断言)`

**关键实践**：
- 使用 `data-testid` 选择器
- 显式等待（`waitForSelector`/`waitForURL`）而非固定延迟
- 内存数据库/事务回滚保证测试隔离
- 失败时截图保存

## 检查清单

场景明确 | 验证点清晰 | 测试独立 | 包含正常+失败 | 速度<10s | 无flaky | CI可运行 | 失败有截图
