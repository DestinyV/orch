# 集成测试提示词

指导实现子代理进行集成测试。优先使用 `../templates/` 中的配置和模板。

## 集成测试目标

验证多组件协作：Repository ↔ Service ↔ Controller | 外部API调用 | 数据库事务 | 缓存交互

## 测试策略

**数据库**：内存数据库优先（SQLite）| 事务回滚保证隔离 | 每个测试独立数据
**外部依赖**：Mock 第三方服务 | 不 Mock 内部模块
**API层**：真实HTTP调用 | 验证请求/响应格式 | 错误码一致

## 关键场景

CRUD完整流程 | 事务边界（成功commit/失败rollback）| 权限校验 | 分页/排序/过滤 | 并发冲突

## 检查清单

多组件协作已验证 | 事务正确 | 数据隔离 | Mock最小化 | 错误处理覆盖 | 性能可接受
