# 异常模式发现指南

编码前扫描项目，发现异常处理约定。核心原则：所有异常类名、错误码格式、RPC 调用模式通过扫描动态发现，禁止硬编码。

## 理论基础

- **约定优于配置**：项目通过已有的异常类、错误码文件、RPC 工具类表达了异常处理约定——新代码应延续而非打破
- **零硬编码原则**：任何预设的异常类名、错误码格式、RPC 调用方式都是对项目约定的猜测，必然有误差。扫描发现是唯一的可靠来源
- **健壮性原理** (Postel's Law)：对所接收的宽容（解析外部响应时容错），对所发送的严格（生成的异常代码遵循项目约定）

## 4 维扫描

### 1. 异常类名与继承关系

扫描目标：项目中异常类定义（`*Exception*`、`*Error*`）和使用方式（`throw new`、`raise`）。不同语言使用对应搜索策略（Java: throws/Exception class, TS: Error class, Python: Exception subclass, Go: errors.New/Err vars）。

记录：异常类名、继承关系、构建方法、import 路径。

### 2. 错误码格式

扫描目标：错误码配置文件（`.properties`、`errors.ts`、`error_codes.py`）和常量定义。识别命名格式模式（如 `MODULE-TYPE-NNN`）和各模块编号范围。

记录：格式模式、文件位置、各模块最大编号。

### 3. RPC 调用模式

扫描目标：HTTP/RPC 客户端（FeignClient、RestTemplate、Axios、fetch）和响应校验方式（`checkRpc`、`HttpTools`、interceptor）。识别远程调用的异常封装模式。

记录：调用方式、响应校验工具类、异常封装类型。

### 4. 模块划分

扫描目标：项目目录结构和模块边界。识别各模块错误码作用域，确保新错误码不跨模块。

记录：模块名称、错误码文件位置、已使用编码范围。

## 输出格式

扫描完成后输出「项目异常规范摘要」：异常类清单（类型+类名+构建方法）、错误码格式+文件位置、RPC 调用方式+校验工具、各模块错误码范围。

## 注意事项

- 扫描仅用于发现，不修改现有约定
- 扫描失败时使用 AskUserQuestion 确认，不猜测
- 所有后续异常代码生成以扫描结果为唯一依据
