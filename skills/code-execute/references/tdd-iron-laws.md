# TDD 铁律

## 核心

**无失败测试不写生产代码。** 写了生产代码再写测试？删除重写，无例外。

## RED → GREEN → REFACTOR

1. **RED**：写一个失败测试，只测一个行为，必须看到红色
2. **GREEN**：写最少代码通过测试，不过度设计
3. **REFACTOR**：测试保持通过，小步重构去重复提可读性

## 规则

- 一测试一行为 | 断言明确 | 命名清晰
- "简单"也先写测试 | 不跳过 TDD | 先实现后补测试 = 删除重写
- 测试不好写 = 设计有问题 | 测试通过后停止，写新测试再做新功能
- 目标：测试驱动设计（覆盖率是副产品）
