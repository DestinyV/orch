# 安全与合规方案

> **适用范围**：所有涉及用户数据、支付、敏感信息或合规要求的项目。
> 本章定义为非必须产出，存在安全合规需求时生成。

## 1. 数据分类

### 1.1 数据分级
| 等级 | 定义 | 示例 | 保护要求 |
|------|------|------|---------|
| L1 公开 | 可对外公开 | 产品名称、公告内容 | 完整性校验 |
| L2 内部 | 仅内部可见 | 业务报表、内部文档 | 访问控制(ACL) |
| L3 敏感 | 泄露有影响 | 手机号、邮箱、地址 | 传输加密+存储加密+访问审计 |
| L4 高度敏感 | 泄露后果严重 | 密码、支付信息、生物特征 | L3要求+字段级加密+脱敏展示 |

### 1.2 PII（个人可识别信息）清单
| 字段 | 所在表/接口 | 等级 | 脱敏策略 | 存储加密 |
|------|-----------|------|---------|---------|
| 手机号 | user.phone | L3 | 138****1234 | AES-256 |
| 邮箱 | user.email | L3 | us***@domain.com | AES-256 |
| 身份证号 | user.id_card | L4 | 110***********1234 | AES-256 |

## 6. 安全运营

### 6.1 漏洞管理
- 扫描工具：[Trivy / Snyk / SonarQube / 商业DAST]
- 扫描频率：依赖扫描[每日] / 代码扫描[每次PR] / 渗透[每季度]
- 修复SLA：高危[7天] / 中危[30天] / 低危[下一个迭代]

### 6.2 安全响应流程
```
发现漏洞 → 定级(P0-P3) → 修复开发 → 安全审查 → 紧急上线 → 复盘文档
```
| 等级 | 定义 | 响应时间 | 修复时间 |
|------|------|---------|---------|
| P0 | 可被远程利用/数据泄露风险 | < 1h | < 24h |
| P1 | 特定条件下的安全风险 | < 4h | < 7天 |
| P2 | 低风险/合规违规 | < 24h | < 30天 |
| P3 | 最佳实践建议 | 下一个迭代 | 下一个迭代 |

### 6.3 安全监控
| 监控项 | 告警条件 | 通知方式 |
|--------|---------|---------|
| 暴力破解 | 同一IP登录失败 > 5次/5min | 钉钉+短信 |
| 异常API访问 | 非正常时序的API调用 | 钉钉 |
| 敏感数据导出 | 大规模数据导出 > 阈值 | 钉钉+审批 |
