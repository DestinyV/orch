# 常用架构模式参考

> code-design 架构决策和 code-execute 实现参考

## Clean Architecture

**核心依赖规则**（内层不依赖外层）：

```
基础设施层(DB/外部API) → 接口适配器层(Controller/Presenter) → 用例层(业务编排) → 实体层(领域模型)
```

**分层职责**：
- 实体层：业务对象和规则
- 用例层：应用特定的业务规则
- 接口适配层：数据格式转换
- 基础设施层：框架、DB、外部服务

## MVC

Model（数据）→ View（UI）→ Controller（交互）。适用于前端和简单 Web 应用。

## 微服务架构

服务拆分（按业务能力）→ 独立部署 → 通过 API/gRPC 通信 → 分布式数据库。

适用场景：大型系统、多团队、需要独立扩展。
