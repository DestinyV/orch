# DDD 基础参考

> design 领域模型设计和 execute 后端实现参考

## 核心概念

| 概念 | 含义 | 示例 |
|------|------|------|
| **限界上下文** | 领域模型的边界 | 订单上下文、用户上下文 |
| **聚合根** | 聚合的入口，唯一外部可访问点 | Order（控制 OrderItem） |
| **实体** | 有唯一标识，关注身份 | User(id)、Order(id) |
| **值对象** | 无标识，关注属性相等 | Address、Money |
| **领域服务** | 跨聚合的业务逻辑 | OrderPricingService |
| **仓储** | 聚合的持久化抽象 | OrderRepository |

## 分层架构

```
表示层(API) → 应用层(用例编排) → 领域层(实体/值对象/领域服务) → 基础设施层(DB/外部API)
```

## 设计原则

- 领域层不依赖基础设施层
- 聚合内部的一致性由聚合根保证
- 值对象不可变
- 仓储只操作聚合根
