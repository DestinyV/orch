# 后端完整性检查清单

> execute 阶段质量审查参考

## 检查项

| # | 检查项 | 要求 |
|---|--------|------|
| 1 | 路由定义 | 所有端点有路径/方法/描述 |
| 2 | 请求校验 | 参数类型/必填/范围校验完整 |
| 3 | 响应结构 | 成功/失败格式统一 |
| 4 | 错误码 | 每个端点定义错误码 + 触发条件 |
| 5 | 数据库操作 | 事务边界明确 + 回滚处理 |
| 6 | 异常处理 | 无空 catch + 无吞掉异常 |
| 7 | 日志 | 关键操作有日志 + 上下文信息 |
| 8 | 性能 | 无 N+1 查询 + 索引覆盖 |
| 9 | 安全 | 无 SQL 注入/XSS/CSRF |
| 10 | 测试 | 每个端点有集成测试 |

## 常见问题

| 问题 | 检测方式 | 修复 |
|------|---------|------|
| 空 catch 块 | `catch.*{.*}` 正则 | 添加日志或重新抛出 |
| SQL 注入 | 字符串拼接 SQL | 使用参数化查询 |
| 未处理空值 | 无 null check | 添加 Optional/防御性编程 |
| 硬编码 | 魔法字符串/数字 | 提取常量或配置 |

## 原则

宁删勿留 | 完整实现 | 不留 TODO | 异常不吞掉 | 日志有意义
