# 数据库设计指南

design 阶段数据库设计参考。

> **Source of Truth**：表设计具体规范见 `table-design-spec.md`（命名/字段/索引/约束/扩展/反模式）。

## 理论基础

- **数据库规范化** (Codd, 1970)：1NF（原子性）→ 2NF（消除部分依赖）→ 3NF（消除传递依赖）。多数场景 3NF 已足够，反规范化仅用于性能优化且需明确标注原因
- **命名约定**：统一小写下划线，表名复数、列名单数。详见 `table-design-spec.md` 第1节
- **索引策略**：覆盖高频查询的 WHERE/JOIN/ORDER BY 列；避免过度索引。详见 `table-design-spec.md` 第3节

## 设计约束

模型根据项目 SQL 方言和业务需求自主设计表结构，遵循：每表必有主键 + created_at/updated_at；外键声明级联策略；金额/数量字段加约束；索引覆盖查询但不冗余。
