# 组件提取指南

design 组件拆分决策和 execute 实现参考。

## 理论基础

- **Atomic Design** (Brad Frost)：原子（基础元素）→ 分子（简单组合）→ 有机体（复杂组件）→ 模板（布局）→ 页面（数据+上下文）。按复杂度层级组织组件
- **单一职责原则（SRP）**：一个组件应只有一个变化理由——当组件同时处理展示逻辑和业务逻辑时，应拆分为容器组件（数据/状态）和展示组件（纯渲染）
- **组合优于继承**：组件通过 props composition 和 slot/children 实现复用，而非继承链
