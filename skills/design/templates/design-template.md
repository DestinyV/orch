# 设计文档模板

```markdown
# [功能名称] 设计方案

**目标**：[功能目标] | **技术栈**：[技术栈] | **相关规范**：[spec/requirement.md]

## 1. 需求分析 - 核心功能、用户场景、技术约束、非功能需求
## 2. 架构设计 - 架构图、模块划分、关键组件、数据流向
## 3. 技术方案 - 技术选型、框架、依赖、API 设计
## 4. 数据库设计（按需）- 数据模型、表结构、索引、迁移
## 5. 接口设计（fullstack）- API 端点、请求/响应格式、错误码
## 6. 核心算法 - 关键业务逻辑伪代码/流程图
## 7. 设计决策 - 决策点 | 方案A | 方案B | 选择 | 原因
## 8. 风险评估 - 技术/性能/兼容性风险 + 缓解措施
## 9. 实施计划 - 分阶段步骤、里程碑、依赖
```
